ABSTRACT

The Smart Construction Management Mobile Application is designed to digitalize and simplify construction site operations. In traditional construction management, tracking materials, approving work progress, and communication between contractors and site engineers is often manual, leading to delays and errors. This application provides a centralized, secure, and intelligent platform for Site Engineers, Contractors, and Supervisors to interact in real-time. Key features include precise material calculators (Brick, Concrete, Steel, Tile), inward/outward inventory management, party-wise ledgers, and automated report generation. The aim of this project is to reduce paperwork, increase transparency, and enhance productivity at construction sites through mobile technology using the Flutter framework.

CHAPTER 1: INTRODUCTION

1.1 Project Summary

The Smart Construction Management App is a mobile-based solution that connects all construction roles in a single system. It allows:

  • Advanced calculators for construction materials.
  • Real-time tracking of inward inventory and stock consumption.
  • Digital management of payment history and dues for stakeholders.
  • One-click professional reporting in PDF/Excel formats.
  • Role-based access-controlled workflow management.

1.2 Aim and Objective

Aim: To develop a mobile application that unifies construction operations into one digital platform.

Objectives:

  • Automate construction material quantity estimation.
  • Replace manual inventory and financial record-keeping.
  • Provide real-time monitoring of site progress and stock levels.
  • Improve transparency and coordination between stakeholders.
  • Maintain a digital archive of project transactions.

1.3 Uses of Application

  • Real-time construction site monitoring.
  • Accurate material requirement planning.
  • Digital financial ledger for project transparency.
  • Professional report generation for audits and meetings.

CHAPTER 4: SYSTEM ANALYSIS

The system analysis identifies how different roles interact and how data flows within the application. Site Engineers use specialized calculators for project estimates, which then feed into the material procurement and inward entry system. Contractors and Supervisors oversee the distribution and consumption of stock, ensuring all transactions are recorded in the party-wise ledgers.

Feasibility studies were conducted to ensure the app is easy to use for field workers with smartphones, requiring minimal training and utilizing local storage for offline reliability.

CHAPTER 5: SYSTEM DESIGN

The system design follows a 3-layer architecture:

  • Presentation Layer: Developed using Flutter Widgets and custom design tokens (Hard Hat Theme).
  • Logic Layer: Provider classes manage role-based control and data synchronization.
  • Data Layer: Shared Preferences and Repository pattern for reliable local storage of construction data.
